# 阿克曼控制器与局部避障使用说明

本文档按当前工程代码重写，说明 `origincar_navigation` 中 Nav2 控制器、共享路径提供器和片段替换式局部避障的实际关系。参数以 `src/origincar_navigation/config/nav2_params.yaml` 当前内容为准。

## 1. 当前导航链路

当前导航行为树是 `behavior_trees/navigate_to_pose_fixed_route_once.xml`，核心策略是：

```text
收到 NavigateToPose 目标
  |
planner_server / GridBased 生成一次全局路径
  |
controller_server / FollowPath 跟踪该路径
  |
TrackingRouteProvider 维护实际跟踪路径 tracking_path
  |
控制器根据 tracking_path 输出 /cmd_vel
```

这里有几个边界要明确：

- 全局规划器只在新目标开始或目标更新时运行，不因为局部障碍反复全局重规划。
- 行为树不调用 Nav2 `SmoothPath`，也不启用官方 recovery、backup、wait、clear costmap 等动作。
- 局部障碍由 controller 内部共享的 `tracking_route_provider` 处理，它只替换当前路径上被挡住的局部片段。
- `local_costmap` 当前使用 `map` 作为 `global_frame`，所以全局路径、避障路径和代价地图处在同一坐标系下，避免把长期路径固化到会漂移的里程计坐标系里。

## 2. 当前可选控制器

`nav2_params.yaml` 的 `controller_server.ros__parameters.controller_plugins` 注册了两个控制器：

```yaml
controller_plugins: ["PurePursuit", "MyPlannerFollowPath"]
```

`navigation.launch.py` 当前默认：

```python
DeclareLaunchArgument('controller_id', default_value='PurePursuit')
```

也就是说，直接启动 `navigation.launch.py` 时默认使用 `PurePursuit`。如果启动时传入 `controller_id:=MyPlannerFollowPath`，行为树里的 `FollowPath` 会改用新控制器。

两个控制器的职责不同，但它们共用同一个局部避障路径提供器：

```text
PurePursuit
  控制算法：OrigincarAckermannPurePursuitController
  参数命名空间：controller_server.PurePursuit

MyPlannerFollowPath
  控制算法：OrigincarAckermannController
  参数命名空间：controller_server.MyPlannerFollowPath

tracking_route_provider
  路径维护/局部避障：SegmentBypassTrackingRouteProvider
  参数命名空间：controller_server.tracking_route_provider
```

因此：

- 调整速度、前视距离、转向限幅时，改对应控制器自己的参数。
- 调整障碍扫描、片段替换、Hybrid A* 搜索时，改 `tracking_route_provider.segment_bypass` 参数。
- 切换控制器不会切换局部避障算法，除非同时修改 `tracking_route_provider.plugin`。

## 3. 控制器和避障的分工

控制器本身不直接修改路径，它只做运动控制：

```text
Nav2 下发 base_plan
  |
controller.setPlan()
  |
tracking_route_provider.resetBasePlan(base_plan)
  |
每个控制周期：
  provider.getTrackingPath(robot_pose)
  |
控制器跟踪 provider 返回的 tracking_path
  |
输出 /cmd_vel
```

`tracking_route_provider` 负责维护实际路径：

- 保存原始 `base_path`。
- 保存当前实际跟踪的 `active_route/tracking_path`。
- 按 `replan_frequency` 低频检查局部障碍。
- 裁剪已经走过的路径点。
- 在 `scan_distance` 范围内查找第一个受阻片段。
- 使用前进-only Hybrid A* 替换受阻片段。
- 发布避障调试路径和状态。

这也是为什么 RViz 中应重点看 `tracking_path`，而不是只看全局规划器输出的路径。真正被控制器跟踪的是 provider 维护后的路径。

## 4. PurePursuit 控制器逻辑

插件类型：

```yaml
plugin: origincar_navigation/OrigincarAckermannPurePursuitController
```

该控制器从 ROS1 参考纯追踪逻辑迁移而来，保留了度制转角、倒车判断和终点附近姿态修正。主要流程是：

```text
取 provider 返回的 tracking_path
  |
判断短前视点是否在车体后方，决定是否倒车
  |
按当前速度计算动态前视距离
  |
选择前视点
  |
用纯追踪公式计算前轮等效转角 delta
  |
按转角和终点距离计算线速度幅值
  |
根据倒车模式给线速度加正负号
  |
用自行车模型输出 cmd_vel.angular.z
```

输出角速度公式：

```text
cmd_vel.angular.z = linear.x * tan(delta) * 0.8 / wheelbase
```

这里的 `delta` 是前轮等效转角，代码中先按度计算，输出前再转成弧度。`0.8` 是当前代码中保留的角速度缩放系数。

### PurePursuit 参数说明

参数位置：

```yaml
controller_server:
  ros__parameters:
    PurePursuit:
```

当前参数快照：

```yaml
min_lookahead_distance: 0.30
max_lookahead_distance: 2.00
lookahead_ratio: 0.50
max_linear_velocity: 0.70
max_angular_: 20.0
goal_tolerance: 0.10
decel_angle_threshold: 15.0
decel_velocity_threshold: 0.20
decel_ratio: 0.40
max_angular_vel: 0.35
wheelbase: 0.143
reverse_lookahead_ratio: 1.00
max_reverse_velocity: 0.45
reverse_check_distance: 0.20
transform_tolerance: 0.20
```

`min_lookahead_distance`：纯追踪最小前视距离，单位 m。低速和停车附近至少会看这么远；太小会让车辆盯着车头附近的点，转向抖动会变大。

`max_lookahead_distance`：纯追踪最大前视距离，单位 m。限制高速或异常速度时前视点不能跳到过远位置；太大可能切弯，太小会跟踪紧但容易抖。

`lookahead_ratio`：前视距离的速度系数。计算关系为 `lookahead = min_lookahead_distance + lookahead_ratio * |current_velocity|`，再受 `max_lookahead_distance` 限制。它控制速度越快时看得多远。

`max_linear_velocity`：前进线速度上限，单位 m/s。控制器计算出来的前进速度幅值不会超过它。

`max_reverse_velocity`：倒车线速度上限，单位 m/s。进入倒车模式时输出负线速度，速度幅值不超过该值。

`reverse_lookahead_ratio`：倒车前视距离缩放系数。倒车时会先按普通公式算前视距离，再乘这个系数；代码还保证结果不低于 `min_lookahead_distance`，避免倒车前视点过近导致角速度打满。

`reverse_check_distance`：倒车判定距离，单位 m。控制器取这个短前视距离处的路径点并转换到 `base_footprint`，如果该点 `x < 0`，认为目标在车后方，本周期进入倒车模式。

`wheelbase`：车辆轴距，单位 m。参与纯追踪转角计算，也参与 `cmd_vel.angular.z` 的自行车模型换算。它必须和底盘/仿真侧使用的几何参数一致，否则同样的转角会被换算成错误角速度。

`max_angular_`：最大前轮等效转角，单位 deg。名字来自旧代码，实际不是 `/cmd_vel.angular.z`，而是纯追踪算出的 `delta` 限幅。

`goal_tolerance`：旧控制器保留参数。当前 Nav2 action 成功主要由 `origincar_goal_checker` 判定，控制器内部停车也会使用传入的 GoalChecker。

`decel_angle_threshold`：转弯减速触发角度，单位 deg。当纯追踪转角绝对值大于该值时，控制器按转角比例降低线速度。

`decel_ratio`：转弯减速强度。越大，大转角时速度压得越低；代码中最小保留 20% 的转弯速度因子。

`max_angular_vel`：旧变量名，当前实际表示转弯时线速度上限，单位 m/s。进入转弯减速分支后，目标线速度还会被该值再限一次。

`decel_velocity_threshold`：单周期速度增加上限，单位 m/s。如果目标速度比当前速度大太多，本周期最多只增加该值，用于抑制突然加速。

`transform_tolerance`：TF 查询超时时间，单位 s。控制器把路径点、目标点转换到 `map` 或 `base_footprint` 时使用；过小会增加 TF 查询失败概率。

## 5. MyPlannerFollowPath 控制器逻辑

插件类型：

```yaml
plugin: origincar_navigation/OrigincarAckermannController
```

该控制器不是传统纯追踪公式，而是按路径进度点和航向误差直接生成速度、角速度：

```text
取 provider 返回的 tracking_path
  |
判断是否倒车
  |
按速度计算前视距离
  |
track_index_ 只向路径后方推进
  |
计算机器人朝向与目标点方向的 yaw_err
  |
中间段：按 yaw_err 降速并输出 yaw_gain * yaw_err
  |
终点段：逐渐从追目标点切换到对齐终点 yaw
  |
按最大前轮转角限制 yaw rate
  |
线速度一阶滤波
  |
输出 /cmd_vel
```

倒车时，它把机器人当前 yaw 加上 180 度作为控制朝向，因此车后方路径点会形成正常的小角度误差，不会强行原地掉头。

### MyPlannerFollowPath 参数说明

参数位置：

```yaml
controller_server:
  ros__parameters:
    MyPlannerFollowPath:
```

当前参数快照：

```yaml
d_min: 0.30
k_gain: 0.50
d_max: 2.00
v_max: 1.0
yaw_gain: 1.50
align_distance: 0.60
terminal_speed_scale: 0.60
dist_tolerance: 0.2
yaw_tolerance: 1.0472
deadband: 0.02
velocity_filter_old_weight: 0.25
wheelbase: 0.143
max_steering_angle_deg: 18.0
reverse_check_distance: 0.40
transform_tolerance: 0.20
```

`d_min`：最小前视距离，单位 m。控制低速时前视目标点离车多远。

`k_gain`：前视距离速度系数。计算关系为 `lookahead = min(d_max, d_min + k_gain * |current_velocity|)`。

`d_max`：最大前视距离，单位 m。限制前视点不会随速度无限增大。

`v_max`：中间路径段线速度上限，单位 m/s。非终点阶段的速度基准来自它。

`yaw_gain`：航向误差到角速度的比例系数。`w_cmd = yaw_gain * yaw_err`，随后还会被转向能力限幅。

`align_distance`：终点姿态融合距离，单位 m。进入最后一个路径点附近该距离内时，控制目标会从“朝向目标点坐标”逐步切换为“对齐终点 yaw”。

`terminal_speed_scale`：终点阶段速度系数。终点附近 `v_cmd = terminal_speed_scale * alpha`，`alpha` 由当前位置到终点距离相对 `align_distance` 计算。

`dist_tolerance`：终点位置容差，单位 m。距离终点小于该值后，控制器记录位置已到达。

`yaw_tolerance`：终点姿态容差，单位 rad。位置到达后，当前代码用 `yaw_err <= yaw_tolerance` 判断姿态是否满足终点条件；这里的 `yaw_err` 是带符号的融合航向误差。

`deadband`：速度和角速度死区。小于该值的线速度或角速度会被置零，避免微小指令让底盘抖动。

`velocity_filter_old_weight`：线速度一阶滤波旧值权重。越大速度变化越慢；`0` 表示完全使用新速度，`1` 会几乎保持旧速度。目标速度为 0 时会直接清空滤波速度。

`wheelbase`：车辆轴距，单位 m。用于把最大前轮转角换算成单位线速度下允许的最大 yaw rate。

`max_steering_angle_deg`：最大前轮转角，单位 deg。控制器换算 `k_w_max = tan(delta_max) / wheelbase`，并用 `|w| <= k_w_max * |v|` 限制角速度。

`reverse_check_distance`：倒车判定距离，单位 m。含义与 `PurePursuit.reverse_check_distance` 一致。

`transform_tolerance`：TF 查询超时时间，单位 s。

## 6. 共享局部避障参数入口

参数位置：

```yaml
controller_server:
  ros__parameters:
    tracking_route_provider:
```

`plugin`：路径提供器插件。当前使用 `origincar_navigation/SegmentBypassTrackingRouteProvider`。如果改成 `origincar_navigation/FixedTrackingRouteProvider`，provider 只发布并返回原始路径，不做片段替换避障。

`enabled`：是否启用片段替换避障。`false` 时 provider 仍维护/发布路径，但不会做障碍扫描和 Hybrid A* 替换。

`replan_frequency`：局部避障更新频率，单位 Hz。只控制 provider 检测障碍和尝试重算绕障片段的频率；控制器本身仍按 `controller_frequency` 高频输出速度。提高它会让障碍响应更快，但会增加 CPU 占用。

`segment_bypass` 下的全部参数控制片段替换避障算法，详见 `路径片段替换式局部避障说明.md`。

## 7. 当前调试话题

话题前缀取决于当前 `controller_id`。默认 `PurePursuit` 时前缀是 `/PurePursuit`；使用新控制器时前缀是 `/MyPlannerFollowPath`。

控制器话题：

```text
/<controller_id>/local_plan
/<controller_id>/target_pose
/<controller_id>/goal_reached
```

`local_plan`：控制器本周期实际用于跟踪的路径。它来自 provider 返回的 `tracking_path`，可能已经包含局部绕障片段。

`target_pose`：本周期控制器选择的前视点或目标点。

`goal_reached`：控制器内部目标到达状态变化时发布。

路径提供器话题：

```text
/<controller_id>/base_path
/<controller_id>/tracking_path
/<controller_id>/blocked_segment
/<controller_id>/bypass_path
/<controller_id>/smoothed_bypass
/<controller_id>/bypass_status
```

`base_path`：Nav2 全局规划器下发给控制器的原始路径。

`tracking_path`：provider 当前维护的实际跟踪路径。无障碍时应基本等于裁剪后的 `base_path`；有障碍且替换成功后，只在受阻片段附近偏离原路径。

`blocked_segment`：本次检测到并准备替换的受阻路径区间，包含前后扩展后的替换范围。

`bypass_path`：Hybrid A* 搜索出来的原始绕障片段。

`smoothed_bypass`：如果启用平滑且路径可用，这里发布平滑后的绕障片段。当前参数文件默认 `smoothing_enabled: false`，所以该话题可能为空。

`bypass_status`：字符串状态，常见内容包括 `active route is clear`、`segment replaced with Hybrid A* bypass`、`segment Hybrid A* failed`、`segment Hybrid A* path is too long` 等。

注意：旧文档中提到的 `astar_path`、`optimized_local_path` 不是当前 provider 源码发布的话题名。

## 8. 启动和切换控制器

默认启动：

```bash
cd /home/xjl/digua
source install/setup.bash
ros2 launch origincar_navigation navigation.launch.py
```

指定新控制器启动：

```bash
ros2 launch origincar_navigation navigation.launch.py controller_id:=MyPlannerFollowPath
```

查看行为树最终使用哪个控制器，可以看 launch 替换后的 `FollowPath controller_id`，也可以观察实际出现的话题前缀：

```bash
ros2 topic list | grep -E 'PurePursuit|MyPlannerFollowPath'
```

## 9. 调参定位建议

发现障碍太晚：优先增大 `tracking_route_provider.segment_bypass.scan_distance`，再考虑提高 `tracking_route_provider.replan_frequency`。

局部避障计算太慢：降低 `replan_frequency`，增大 `astar_grid_downsample_factor`，减少 `hybrid_yaw_bins`，或减小 `scan_distance`。

绕障路径离障碍太近：检查 `local_costmap.robot_radius`、`local_costmap.inflation_layer.inflation_radius` 和 `astar_lethal_cost`。避障算法只相信 local costmap 里的代价，安全距离首先来自代价地图。

绕障绕出大圆：减小 `hybrid_search_lateral_margin` 或 `hybrid_max_path_length_ratio`，必要时降低 `hybrid_goal_yaw_weight`，让终点 yaw 不主导局部搜索。

U 形弯里路径跳到另一侧：减小 `prune_search_distance`，限制机器人最近点搜索只看当前路径前方局部范围。

绕障连接处不顺：增大 `pre_margin/post_margin` 或 `pre_point_margin/post_point_margin`，给 Hybrid A* 和后续跟踪留下更长过渡段；如果启用平滑，还要开启 `smoothing_collision_check` 防止平滑后压进障碍。

车辆转向不够：对 `PurePursuit` 检查 `max_angular_`、`wheelbase` 和底盘对 `/cmd_vel.angular.z` 的解释；对 `MyPlannerFollowPath` 检查 `max_steering_angle_deg`、`wheelbase`、`yaw_gain`。

倒车逻辑异常：检查 `reverse_check_distance` 是否过短或过长，并观察短前视点转换到 `base_footprint` 后是否确实落在车体后方。
