# 路径片段替换式局部避障说明

本文档专门说明当前 `SegmentBypassTrackingRouteProvider` 和 `SegmentBypassPlanner` 的局部避障逻辑。该算法不是全局重规划，也不是滚动窗口生成一整条新局部轨迹，而是在当前跟踪路径上找到被障碍挡住的短片段，然后只替换这段路径。

## 1. 核心对象

`base_path`

Nav2 全局规划器下发给控制器的原始路径。新目标或新路线到来时，控制器 `setPlan()` 会把它交给 `tracking_route_provider.resetBasePlan()`。

`active_route / tracking_path`

控制器实际跟踪的当前路径。初始化时等于 `base_path`；机器人前进后会裁剪掉已走过的点；局部障碍替换成功后，它会包含绕障片段。控制器每个周期拿到的就是这条路径。

`blocked_segment`

在当前 `tracking_path` 前方扫描到的受阻片段。发布出来的 `blocked_segment` 不是只有碰撞点本身，而是经过前后 margin 扩展后的待替换区间。

`bypass_path`

前进-only Hybrid A* 搜索得到的原始绕障片段。它从替换区间起点附近的可通行栅格出发，连接到替换区间终点附近的可通行栅格。

`smoothed_bypass`

对 `bypass_path` 做轻量三点平滑后的结果。当前参数文件默认关闭平滑，因此实际替换通常直接使用 `bypass_path`。

## 2. 完整处理流程

```text
getTrackingPath(robot_pose)
  |
按 replan_frequency 判断本周期是否需要更新避障
  |
检查 active_route frame 是否等于 local_costmap frame
  |
获取 local_costmap 并加锁
  |
SegmentBypassPlanner.updateRoute()
  |
  1. 裁剪 active_route 已走过部分
  2. 按 route_sample_interval 重采样加密路径
  3. 在 scan_distance 内查找第一个受阻片段
  4. 按距离和点数扩展替换区间
  5. 寻找替换起点/终点附近可通行栅格
  6. 判断替换区间长度是否足够
  7. 运行前进-only Hybrid A*
  8. 检查绕障路径长度是否过长
  9. 可选平滑并可选碰撞复查
 10. 用绕障片段替换 active_route 对应区间
  |
替换成功则更新 active_route
  |
发布 tracking_path、blocked_segment、bypass_path、smoothed_bypass、bypass_status
```

如果任一步失败，provider 不会把失败路径写入 `active_route`。它会继续返回上一条可用路径，并通过 `bypass_status` 发布失败原因。

## 3. 路径裁剪逻辑

每次重算前，算法先调用 `pruneRouteFromRobot()`：

```text
从 active_route 开头向后累计 prune_search_distance
  |
只在这段局部范围内找离机器人最近的路径点
  |
从最近点开始保留后续路径
  |
重算中间路径点 yaw
  |
恢复最后一个 goal 点的原始 orientation
```

这里有两个关键设计：

- 最近点搜索不扫描整条路径，只扫描前方 `prune_search_distance` 范围，避免 U 形路线中跳到另一侧路径臂。
- 裁剪后不会把机器人当前位置插入路径首点。机器人当前位置到最近路径点之间的短线不一定是原始路径，可能穿过障碍膨胀层；插进去会制造假的受阻片段。

## 4. 路径重采样逻辑

裁剪后，算法用 `route_sample_interval` 对当前路径折线重采样：

```text
active_route 点距可能不均匀
  |
按固定间隔插入路径点
  |
只改变点密度，不改变几何形状
  |
重新计算中间点 yaw
  |
恢复终点原始 orientation
```

重采样的目的是提高障碍检测和替换区间选择的稳定性。如果原始路径点太稀，障碍可能只压住一个点，导致替换片段过短，Hybrid A* 起终点离障碍太近。

终点 orientation 必须恢复，因为控制器终点附近会读取最后一个路径点的 yaw 来做虚拟前视点和停车姿态控制。如果把终点 yaw 改成最后一小段折线切线，倒车和终点对齐可能出错。

## 5. 受阻片段检测

`findFirstBlockedSegment()` 沿当前路径从近到远扫描：

```text
只扫描 scan_distance 距离
  |
对每段路径按 costmap 分辨率一半采样
  |
采样点落入 local_costmap 且 cost >= astar_lethal_cost
  |
记录连续受阻 run
  |
连续受阻长度 >= min_blocked_length 才认为有效
```

未知空间的处理要注意：

- `isWorldPointBlocked()` 中，`NO_INFORMATION` 不会被当成阻塞点。
- Hybrid A* 搜索中，未知栅格是否可通行由 `astar_allow_unknown` 决定。

也就是说，路径扫描阶段不会因为未知空间直接报 blocked；真正搜索时是否允许穿越未知，要看 `astar_allow_unknown`。

## 6. 替换区间扩展

检测到受阻 run 后，算法不会只替换障碍覆盖的点，而是扩展成更长的替换区间：

```text
blocked first/last index
  |
向前扩展 pre_margin 米
  |
向后扩展 post_margin 米
  |
再向前扩展 pre_point_margin 个路径点
  |
再向后扩展 post_point_margin 个路径点
```

距离扩展负责让起终点离障碍有物理距离；点数扩展负责补偿采样不均匀问题。两者一起使用可以避免“障碍只占住几个密集点或稀疏点，替换段太短”的问题。

扩展后还会调用 `findTraversableRouteCell()`：

- 起点如果落在障碍里，就沿路径向前搜索可通行点。
- 终点如果落在障碍里，就沿路径向后搜索可通行点。
- 找不到可通行起终点时，本次替换失败，状态为 `failed to find free segment endpoints on route`。

## 7. Hybrid A* 搜索逻辑

当前 Hybrid A* 是局部、前进-only、阿克曼运动学搜索：

```text
状态：x 栅格、y 栅格、yaw 离散桶
动作：固定步长 step_distance + 不同前轮转角
方向：只向前，不生成倒车动作
碰撞：检查整段圆弧中间采样点，不只检查终点
终点：到达目标位置即可，终点 yaw 只作为软代价
范围：限制在起点到终点连线附近的局部走廊内
长度：超过原替换段长度倍率则拒绝
```

不把终点 yaw 作为硬约束是当前实现的重要点。局部绕障片段通常很短，如果强制车头到达终点时完全对齐目标 yaw，车辆可能为了调正朝向绕出一个大半圆。当前代码只把 yaw 误差放进 heuristic 软代价。

搜索过程中还会做两层可通行判断：

- 动作圆弧上的采样点必须在 costmap 内，且原始栅格可通行。
- 节点所在的降采样栅格必须整块可通行，避免降采样后从障碍缝隙里穿过去。

## 8. 平滑和回退逻辑

如果 `smoothing_enabled: true` 且 `bypass_path` 至少有 4 个点，会执行轻量平滑：

```text
取 bypass_path 点集
  |
临时加入替换区间前后相邻锚点
  |
迭代 4 次三点平滑
  |
删除临时锚点
  |
按 sample_interval 重采样
```

平滑只移动绕障片段内部点，不移动真正替换区间首尾连接点。

如果 `smoothing_collision_check: true`，平滑后会重新沿路径采样检查碰撞：

- 平滑路径无碰撞：使用 `smoothed_bypass` 替换原路径。
- 平滑路径为空或碰撞：回退使用原始 `bypass_path`。

如果 Hybrid A* 本身失败，则不会替换路径。

### 9.1 provider 开关参数

`plugin`

控制使用哪种路径提供器。`SegmentBypassTrackingRouteProvider` 表示启用“维护 active_route + 片段替换避障”的 provider；`FixedTrackingRouteProvider` 表示始终返回原始路径。

`enabled`

控制 `SegmentBypassPlanner.updateRoute()` 是否真正执行避障。为 `false` 时，本模块不会扫描障碍或替换路径。

`replan_frequency`

避障更新频率，单位 Hz。影响 `maybeUpdateRoute()` 多久进入一次局部重算。它不改变 controller 的速度输出频率；只改变障碍检测和路径替换频率。

### 9.2 扫描和裁剪参数

`scan_distance`

沿当前 `tracking_path` 向前扫描障碍的最大距离，单位 m。越大越早发现远处障碍，但计算更多，也可能过早处理还不相关的障碍。

`prune_search_distance`

从当前路径开头向后累计的最近点搜索范围，单位 m。它控制裁剪路径时最多往前看多远找“机器人当前所在路径进度”。U 形路径或近距离平行路径中，该值过大会跳到另一条路径臂；过小则可能裁剪不掉已走过的点。

`route_sample_interval`

检测受阻片段前，对当前路径重采样的间隔，单位 m。越小路径点越密，障碍检测和替换区间更稳定，但点数更多。代码下限按 `0.02m` 保护。

`min_blocked_length`

连续受阻长度阈值，单位 m。小于该长度的连续命中会被当作噪声忽略。增大它可以减少误触发；减小它可以更敏感地响应小障碍。

### 9.3 替换区间参数

`pre_margin`

从受阻片段起点向路径前方回退扩展的距离，单位 m。它控制绕障起点离障碍有多远。

`post_margin`

从受阻片段终点向路径后方继续扩展的距离，单位 m。它控制绕障终点接回原路径时离障碍有多远。

`pre_point_margin`

在 `pre_margin` 扩展之后，替换起点再额外向前移动的路径点数量。它不按米计算，而按重采样后的点数计算。

`post_point_margin`

在 `post_margin` 扩展之后，替换终点再额外向后移动的路径点数量。用于给绕障段更自然的接回空间。

`min_replace_length`

最终替换区间的最小长度，单位 m。短于该值时不做 Hybrid A*，状态为 `blocked segment is too short to replace`。该参数用于避免极短片段生成不自然绕行曲线。

### 9.4 costmap 和 A* 代价参数

`astar_allow_unknown`

Hybrid A* 是否允许穿过 `NO_INFORMATION` 未知栅格。`false` 更保守；`true` 可以在局部代价地图未知区域更多时提高可达性，但风险更高。

`astar_lethal_cost`

代价值大于等于该值的栅格视为不可通行。它同时影响受阻检测和搜索碰撞判断。降低该值会让膨胀层中较低代价也被视为障碍，路径更保守；升高则更贴近障碍。

`astar_cost_weight`

costmap 归一化代价进入 A* `g_score` 的权重。越大，搜索越倾向避开高代价区域；越小，路径更接近几何最短和转向代价。

`astar_grid_downsample_factor`

Hybrid A* 搜索网格降采样倍率。`1` 表示原始 costmap 分辨率；`2` 表示搜索栅格边长扩大 2 倍，节点数量约降为四分之一。降采样越大计算越快，但路径精细度下降。

### 9.5 Hybrid A* 车辆模型参数

`hybrid_wheelbase`

Hybrid A* 内部车辆轴距，单位 m。用于模拟阿克曼圆弧。应与控制器 `wheelbase` 和底盘模型保持一致。

`hybrid_max_steering_angle`

Hybrid A* 运动原语最大前轮转角，单位 rad。它决定搜索时能生成多小转弯半径。

`hybrid_yaw_bins`

yaw 离散桶数量。越大，朝向状态越细，路径更细腻，但搜索空间更大。代码至少使用 8 个桶。

`hybrid_steering_samples`

每个节点展开的转角采样数量。当前只采样前进动作，例如 7 表示从最大左转到最大右转之间取 7 个转角。越大可选动作越多，计算也越重。

`hybrid_step_distance`

每个动作前进距离，单位 m。步长小，搜索更细但节点更多；步长大，搜索快但可能跨过窄通道或路径粗糙。代码会保证不小于 costmap 分辨率和降采样尺度对应的下限。

`hybrid_goal_tolerance`

到达替换终点的位置容忍度，单位 m。当前只要位置进入该容差，或落入目标栅格，就认为搜索到达。

`hybrid_yaw_tolerance`

终点 yaw 软代价死区，单位 rad。yaw 误差小于该值时不额外惩罚；超过后才按误差加入 heuristic。

`hybrid_max_iterations`

单次搜索最大展开次数。太小可能找不到可行路径；太大会在复杂障碍中占用更多时间。

`hybrid_steering_cost_weight`

转角代价权重。越大，搜索越不喜欢大转角，路径会更平缓，但可能更难绕过近距离障碍。

`hybrid_goal_yaw_weight`

终点 yaw 软代价权重。越大越倾向以接近终点 yaw 的朝向到达；过大可能重新诱发大圆绕行。当前它不是硬约束。

`hybrid_max_path_length_ratio`

绕障路径相对原替换区间长度的最大倍率。超过 `reference_replace_length * hybrid_max_path_length_ratio` 的结果会被拒绝，避免局部绕障跑成全局大绕行。

`hybrid_search_lateral_margin`

局部搜索走廊半宽，单位 m。搜索状态必须靠近替换起点到终点连线，超过该距离会被丢弃。它限制绕障不要跑出局部区域；太小会找不到绕过障碍的空间。

### 9.6 输出和平滑参数

`sample_interval`

`bypass_path` 和 `smoothed_bypass` 输出重采样间隔，单位 m。越小输出路径越密，控制器跟踪更连续，但点数更多。

`smoothing_enabled`

是否启用轻量三点平滑。当前参数文件默认 `false`，所以默认直接使用 Hybrid A* 原始路径替换。

`smoothing_collision_check`

平滑后是否重新检查碰撞。只有 `smoothing_enabled` 为 `true` 时才有意义。建议启用平滑时也启用该项，否则平滑可能把路径推入障碍膨胀层。

## 10. 状态和调试话题

话题前缀等于当前控制器 ID。默认启动时是 `/PurePursuit`。

```text
/<controller_id>/base_path
/<controller_id>/tracking_path
/<controller_id>/blocked_segment
/<controller_id>/bypass_path
/<controller_id>/smoothed_bypass
/<controller_id>/bypass_status
```

重点判断方式：

- `base_path` 正常但 `tracking_path` 异常：问题多在 provider 裁剪、替换或坐标系。
- `blocked_segment` 为空且 `bypass_status` 为 `active route is clear`：当前扫描范围内没有检测到被 `astar_lethal_cost` 判为阻塞的路径点。
- `blocked_segment` 有内容但 `bypass_path` 为空：通常是 Hybrid A* 未找到路径、起终点不可通行或搜索范围太窄。
- `bypass_path` 有内容但 `tracking_path` 没变：检查 `bypass_status` 是否提示路径过长或替换失败。
- `smoothed_bypass` 为空：如果 `smoothing_enabled: false`，这是正常现象。

常见 `bypass_status`：

`segment bypass disabled`：`enabled` 为 `false`。

`active route is clear`：扫描范围内路径未受阻。

`failed to find free segment endpoints on route`：扩展后的替换起点或终点仍找不到可通行栅格。

`blocked segment is too short to replace`：替换区间长度小于 `min_replace_length`。

`segment Hybrid A* failed`：搜索在约束范围内未找到可行前进路径。

`segment Hybrid A* path is too long`：搜索有结果，但长度超过 `hybrid_max_path_length_ratio` 限制。

`segment replaced with Hybrid A* bypass`：使用原始 Hybrid A* 路径完成替换。

`segment replaced with smoothed bypass`：平滑路径可用，并用平滑结果完成替换。

## 11. 推荐验证步骤

无障碍路径：

- `tracking_path` 应等于裁剪和重采样后的原路径，不应突然跨到其他路线。
- `bypass_status` 应主要是 `active route is clear`。
- 控制器 `local_plan` 应与 `tracking_path` 一致。

静态障碍：

- 障碍进入 `scan_distance` 后，`blocked_segment` 应显示被扩展后的替换段。
- `bypass_path` 应绕开 local costmap 中 `cost >= astar_lethal_cost` 的区域。
- `tracking_path` 只在障碍附近被替换，后续路线仍接回原路径。
- `planner_server` 不应因为这个局部障碍重新生成全局路径。

U 形或近距离平行路线：

- 观察路径裁剪后的 `tracking_path` 是否跳到另一侧路径臂。
- 如果跳臂，优先减小 `prune_search_distance`。
- 不要通过增大 `scan_distance` 解决跳臂问题，因为跳臂发生在路径裁剪阶段，不是障碍扫描距离不足。

平滑验证：

- 先在 `smoothing_enabled: false` 下确认 Hybrid A* 原始路径可用。
- 再打开 `smoothing_enabled: true` 和 `smoothing_collision_check: true`。
- 如果平滑后路径贴障碍或失败，保留碰撞复查，让算法自动回退到 `bypass_path`。
