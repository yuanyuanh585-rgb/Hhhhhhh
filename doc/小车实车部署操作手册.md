# 小车实车部署操作手册

本文档用于把当前导航算法部署到真实 Origincar 小车。当前工程采用一套参数文件，实车默认使用系统时间；仿真启动入口会显式切到 Gazebo `/clock`。

## 1. 软件依赖

建议环境：

```bash
source /opt/ros/humble/setup.bash
sudo apt update
sudo apt install -y \
  python3-colcon-common-extensions \
  python3-rosdep \
  ros-humble-ament-lint-auto \
  ros-humble-ament-lint-common \
  ros-humble-navigation2 \
  ros-humble-nav2-bringup \
  ros-humble-nav2-map-server \
  ros-humble-slam-toolbox \
  ros-humble-robot-localization \
  ros-humble-imu-filter-madgwick \
  ros-humble-tf2-ros \
  ros-humble-teleop-twist-keyboard \
  ros-humble-rviz2 \
  ros-humble-pcl-conversions \
  libpcap-dev \
  libpcl-dev
```

首次使用 `rosdep`：

```bash
sudo rosdep init
rosdep update
cd /home/xjl/digua
rosdep install --from-paths src --ignore-src -r -y
```

## 2. 编译

```bash
cd /home/xjl/digua
source /opt/ros/humble/setup.bash
colcon build --packages-select \
  origincar_msg \
  origincar_base \
  lslidar_msgs \
  lslidar_driver \
  origincar_mapping \
  origincar_navigation \
  origincar_strategy \
  origincar_bringup
source install/setup.bash
```

## 3. 设备检查

当前代码默认：

- 底盘串口：`/dev/ttyACM0`
- N10 雷达串口：`/dev/ttyACM1`
- 雷达话题：`/scan`
- 雷达坐标系：`laser_link`
- 里程计融合坐标系：`odom_combined`

检查设备：

```bash
ls -l /dev/ttyACM*
groups
```

如果当前用户不在 `dialout` 组：

```bash
sudo usermod -aG dialout $USER
```

重新登录后生效。

## 4. 一键启动实车底盘和传感器

```bash
cd /home/xjl/digua
source install/setup.bash
ros2 launch origincar_bringup real_robot.launch.py
```

该入口只启动实车导航闭环需要的节点：底盘串口、N10 雷达、必要静态 TF、IMU 滤波和 EKF。不默认启动 RViz、joint_state_publisher 或仿真节点。

检查基础数据：

```bash
ros2 topic echo /scan --once
ros2 topic echo /odom --once
ros2 topic echo /odom_combined --once
ros2 run tf2_ros tf2_echo odom_combined base_footprint
ros2 run tf2_ros tf2_echo base_footprint laser_link
```

## 5. 实车建图

启动建图：

```bash
ros2 launch origincar_bringup real_mapping.launch.py
```

另开终端遥控小车低速移动：

```bash
source /home/xjl/digua/install/setup.bash
ros2 run teleop_twist_keyboard teleop_twist_keyboard
```

保存地图：

```bash
ros2 launch origincar_bringup real_save_map.launch.py
```

保存成功后应生成：

```text
maps/real/map.yaml
maps/real/map.pgm
```

地图保存后，需要基于新地图重新采集路线点，并写入：

```text
maps/real/global_routes.yaml
```

## 6. 实车导航

启动导航：

```bash
ros2 launch origincar_bringup real_navigation.launch.py
```

调试时单独打开 RViz：

```bash
ros2 launch origincar_bringup real_view_navigation.launch.py
```

如确实希望导航启动时一起打开 RViz：

```bash
ros2 launch origincar_bringup real_navigation.launch.py use_rviz:=true
```

实车导航默认读取：

```text
/home/xjl/digua/maps/real/map.yaml
/home/xjl/digua/maps/real/global_routes.yaml
```

临时调试其他地图时，需要地图和路线文件一起覆盖：

```bash
ros2 launch origincar_bringup real_navigation.launch.py map:=/path/to/map.yaml routes_file:=/path/to/global_routes.yaml
```

检查 Nav2 生命周期：

```bash
ros2 lifecycle get /planner_server
ros2 lifecycle get /controller_server
ros2 lifecycle get /bt_navigator
```

正常应为 `active [3]`。

## 7. 仿真入口

仿真入口使用 `sim_` 前缀区分，仿真会使用 Gazebo `/clock`：

```bash
ros2 launch origincar_bringup sim.launch.py
ros2 launch origincar_bringup sim_mapping.launch.py
ros2 launch origincar_bringup sim_navigation.launch.py
```

实车入口使用 `real_` 前缀，仿真入口使用 `sim_` 前缀。

## 8. 常见排查

没有 `/scan`：

```bash
ros2 node list | grep lslidar
ros2 param get /lslidar_driver_node serial_port_
ros2 param get /lslidar_driver_node frame_id
```

没有 `/odom_combined`：

```bash
ros2 node list | grep ekf
ros2 topic echo /odom --once
ros2 topic echo /imu/data_raw --once
```

TF 不连通：

```bash
ros2 run tf2_ros tf2_echo map odom_combined
ros2 run tf2_ros tf2_echo odom_combined base_footprint
ros2 run tf2_ros tf2_echo base_footprint laser_link
```

导航不动：

```bash
ros2 topic echo /cmd_vel --once
ros2 topic echo /plan --once
ros2 topic echo /laser_localizer/score --once
```

如果 `/cmd_vel` 有输出但底盘不动，优先检查底盘串口权限、电源、急停和 `/dev/ttyACM0` 是否被其他进程占用。
