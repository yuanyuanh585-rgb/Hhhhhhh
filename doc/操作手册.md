## 2. 项目运行命令

### 2.1 编译项目

每次修改代码后需要重新编译。

**编译全部包：**

```bash
source /opt/ros/humble/setup.bash
colcon build
source install/setup.bash
```

**按包选编（推荐，速度更快）：**

```bash
source /opt/ros/humble/setup.bash
colcon build --packages-select \
  origincar_msg \
  origincar_base \
  lslidar_msgs \
  lslidar_driver \
  origincar_mapping \
  origincar_navigation \
  origincar_strategy \
  origincar_bringup
source install/setup.bash
```

> **提示：** 建议在 `~/.bashrc` 中添加以下两行，每次新开终端自动加载环境：
> ```bash
> source /opt/ros/humble/setup.bash
> source install/setup.bash
> ```

### 2.2 启动实车底盘和传感器

该命令只启动底层硬件驱动和传感器，不启动建图/导航算法：

```bash
source install/setup.bash
ros2 launch origincar_bringup real_robot.launch.py
```

启动内容：

| 组件 | 说明 |
|------|------|
| `origincar_base` | 底盘串口驱动（`/dev/ttyACM0`，波特率 115200） |
| `lslidar_driver` | N10 激光雷达驱动（发布 `/scan`） |
| `imu_calib` | IMU 校准节点 |
| `imu_complementary_filter` | IMU 互补滤波（发布 `/imu/data`） |
| `ekf_filter_node` | EKF 融合里程计与 IMU（发布 `odom_combined`） |
| 静态 TF | `base_footprint → base_link → gyro_link → laser_link` |

**可用参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `use_sim_time` | `false` | 实车使用系统时间，不可改为 true |
| `enable_ekf` | `true` | 是否启用 EKF 节点 |
| `publish_odom_tf` | `true` | 底盘是否发布 odom TF |

### 2.3 实车建图

**步骤一：启动实车底盘 + SLAM 建图**

```bash
source install/setup.bash
ros2 launch origincar_bringup real_mapping.launch.py
```

该命令自动启动 `real_robot.launch.py`（底盘+传感器+EKF），并在 3 秒后启动 `slam_toolbox` 在线建图。

**可用参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `use_sim_time` | `false` | 实车使用系统时间 |
| `params_file` | `slam_toolbox.yaml` | SLAM 参数文件路径 |

**步骤二：键盘遥控小车移动建图**

另开终端：

```bash
source install/setup.bash
ros2 run teleop_twist_keyboard teleop_twist_keyboard
```

常用按键：

```
i  前进      ,  后退
j  左转      l  右转
k  停止
q/z  增减最大速度    w/x  增减线速度    e/c  增减角速度
```

**步骤三：保存地图**

```bash
source install/setup.bash
ros2 launch origincar_bringup real_save_map.launch.py
```

默认保存到 `maps/real/map.yaml` 和 `maps/real/map.pgm`。可指定路径：

```bash
ros2 launch origincar_bringup real_save_map.launch.py map_name:=maps/real/my_map
```

建图建议：

- 低速行驶，避免高速急转
- 沿场地围挡绕一圈，再覆盖中间区域
- 地图边界逐渐变清晰后再保存

### 2.4 实车导航

**前提条件：** 已完成建图并保存地图文件，同时已配置好路线文件。

```bash
source install/setup.bash
ros2 launch origincar_bringup real_navigation.launch.py  
```

```bash
source install/setup.bash
# 启用杭电的控制算法
ros2 launch origincar_bringup real_navigation.launch.py controller_id:=MyPlannerFollowPath
```

启动内容：

| 组件 | 说明 |
|------|------|
| `real_robot.launch.py` | 底盘 + 雷达 + EKF |
| `map_server` | 加载地图，发布 `/map` |
| `laser_predictive_localizer` | 纯激光预测定位（发布 `map → odom_combined`） |
| `navigation.launch.py` | Nav2 规划/控制/行为树 |
| `route_strategy` | 路线策略控制器 |

**可用参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `map` | `maps/real/map.yaml` | 地图 YAML 文件路径 |
| `routes_file` | `maps/real/global_routes.yaml` | 路线配置文件 |
| `params_file` | `nav2_params.yaml` | Nav2 参数文件 |
| `localizer_params_file` | `laser_predictive_localizer.yaml` | 定位参数文件 |
| `strategy_params_file` | `strategy_params.yaml` | 策略参数文件 |
| `use_sim_time` | `false` | 实车使用系统时间 |
| `use_rviz` | `false` | 是否同时启动 RViz |

**更换地图和路线：**

```bash
ros2 launch origincar_bringup real_navigation.launch.py \
  map:=/path/to/your_map.yaml \
  routes_file:=/path/to/your_routes.yaml
```

**带 RViz 启动导航：**

```bash
ros2 launch origincar_bringup real_navigation.launch.py use_rviz:=true
```

### 2.5 查看导航可视化

启动 RViz 查看导航可视化（需先在另一个终端中运行导航）：

```bash
source install/setup.bash
ros2 launch origincar_bringup real_view_navigation.launch.py
```

该入口启动 RViz 并加载 `rviz_nav.rviz` 配置文件，同时包含路线点 Marker 可视化。

**可用参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `routes_file` | `maps/real/global_routes.yaml` | 路线文件 |
| `use_route_markers` | `true` | 是否显示路线 Marker |

**在 RViz 中手动添加显示项：**

| 显示类型 | Topic | 说明 |
|----------|-------|------|
| Map | `/map` | 显示占据栅格地图 |
| LaserScan | `/scan` | 显示激光雷达扫描 |
| TF | — | 显示各坐标系 |
| RobotModel | — | 显示机器人 3D 模型 |
| Path | `/plan` | 显示全局规划路径 |
| MarkerArray | `/global_route_markers` | 显示预设路线点标记 |
| Odometry | `/odom_combined` | 显示融合里程计 |

**在 RViz 中操作：**
- 使用 **2D Pose Estimate** 设置初始位姿（给定位节点提供初始值）
- 使用 **Nav2 Goal** 或 **2D Goal Pose** 发送目标点

## 3. 点位设置功能

### 3.1 路线配置文件说明

路线文件默认位置：

- **实车路线：** `maps/real/global_routes.yaml`
- **导航包内置：** `src/origincar_navigation/config/global_routes.yaml`
- **策略包内置：** `src/origincar_strategy/config/global_routes.yaml`

**文件格式：**

```yaml
routes:
  route_1:
    points:
      - x: -1.696
        y: -2.271
        orientation: {x: 0.0, y: 0.0, z: 0.006442, w: 0.999979}
      - x: 1.94
        y: -1.21
        orientation: {x: 0.0, y: 0.0, z: 0.3906, w: 0.9205}

  route_2:
    points:
      - x: -0.002
        y: -0.491
        orientation: {x: 0.0, y: 0.0, z: 0.7026, w: 0.7115}
      # ... 更多点位

  route_3:
    points:
      # ...

  route_4:
    points:
      # ...
```

**字段说明：**

| 字段 | 说明 |
|------|------|
| `routes` | 路线集合，最多支持 4 条路线（`route_1` ～ `route_4`） |
| `points` | 路线的点位列表，按从前到后顺序排列 |
| `x`, `y` | 点在 `map` 坐标系下的坐标 |
| `orientation` | 点位的朝向，四元数格式 `{x, y, z, w}` |
| `yaw_deg`（可选） | 角度制朝向，如 `yaw_deg: 90.0` 表示朝 y 正方向 |

> **提示：** 修改路线文件后，需要重启导航才能使新路线生效。

### 3.2 使用点位格式化工具采集点位

项目提供了 `route_point_formatter.py` 工具，可以将 RViz 中点击的位姿自动格式化为可直接粘贴到路线文件的 YAML 格式。


**启动图文贴图，方便观测**
```bash
source install/setup.bash
ros2 launch origincar_sim texture_marker.launch.py \
  map_yaml_path:=/root/digua/maps/real/map.yaml
```

**监听 InitialPose（推荐，更稳定）：**

```bash
source install/setup.bash
ros2 run origincar_navigation route_point_formatter.py --ros-args -p topic:=/initialpose
```

然后在 RViz 中点击 **2D Pose Estimate**，终端会输出格式化后的点位：

```yaml
# from /initialpose
- x: -1.230000
  y: 0.450000
  orientation: {x: 0.000000, y: 0.000000, z: 0.382683, w: 0.923880}
  # yaw_deg: 45.000000
```

直接将前三行复制到对应的路线 `points` 列表中即可。

**监听 GoalPose：**

```bash
ros2 run origincar_navigation route_point_formatter.py --ros-args -p topic:=/goal_pose
```

**单次模式（接收一次后自动退出）：**

```bash
ros2 run origincar_navigation route_point_formatter.py --ros-args -p topic:=/initialpose -p once:=true
```

**原始方法（不使用格式化工具）：**

```bash
# 监听初始位姿
ros2 topic echo /initialpose --once

# 监听目标位姿
ros2 topic echo /goal_pose --once
```

然后在输出中手动提取 `position.x/y` 和 `orientation` 填入路线文件。

**采集流程建议：**

1. 启动导航系统（底盘 + 地图 + 定位）
2. 启动 RViz 查看地图
3. 运行 `route_point_formatter.py` 监听 `/initialpose`
4. 在 RViz 中用 **2D Pose Estimate** 在地图上依次点击路径点
5. 将终端输出的点位复制到 `global_routes.yaml` 对应路线中
6. 重启导航使新路线生效

### 3.3 路线点可视化渲染查看

**在导航启动时查看路线点：**

项目提供了路线点可视化节点，可将 `global_routes.yaml` 中的点位渲染为 RViz 中的彩色小球和方向箭头。

**独立启动路线可视化（不需要完整的导航系统）：**

```bash
source install/setup.bash
ros2 launch origincar_navigation route_markers.launch.py
```

**指定路线文件：**

```bash
ros2 launch origincar_navigation route_markers.launch.py \
  routes_file:=maps/real/global_routes.yaml
```

**启动后，在 RViz 中添加显示：**

1. 将 `Fixed Frame` 设置为 `map`
2. 点击 **Add** → 选择 **MarkerArray**
3. Topic 选择 `/global_route_markers`

**显示效果：**
- **彩色小球：** 每条路线的点位位置（不同路线颜色不同）
- **方向箭头：** 每个点位朝向的方向指示

**可用参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `routes_file` | `global_routes.yaml` | 路线配置文件路径 |
| `frame_id` | `map` | Marker 参考坐标系 |
| `marker_topic` | `global_route_markers` | 发布 Marker 的话题名 |
| `publish_period_ms` | `1000` | Marker 发布周期（毫秒） |

**使用 view_navigation 入口查看路线（同时打开 RViz + 路线 Marker）：**

```bash
ros2 launch origincar_bringup real_view_navigation.launch.py
```

## 4. 策略启动与导航触发

### 4.1 策略控制器说明

`origincar_strategy` 包提供了路线策略控制器 (`route_strategy_controller`)，负责根据机器人在全局路线上的位置，自动切换分支路线。它通过调用 Nav2 的 `NavigateToPose` Action 按预设路线依次导航。

**策略工作流程：**

```
route_1 → 等待切换 → 分支路线(route_2 或 route_3) → 过渡 → route_4
```

**策略包独立启动（一般不单独使用，由导航入口自动包含）：**

```bash
source install/setup.bash
ros2 launch origincar_strategy route_strategy.launch.py
```

**策略参数说明（`strategy_params.yaml`）：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `waiting_switch_timeout` | `2.0` | 等待路线切换的超时时间（秒） |
| `default_branch_route` | `2` | 默认分支路线编号 |
| `transition_back_distance` | `0.45` | 倒车过渡距离（米） |
| `transition_skip_angle_threshold_deg` | `65.0` | 跳过大角度过渡的阈值（度） |
| `global_frame` | `map` | 全局坐标系 |
| `robot_base_frame` | `base_footprint` | 机器人底盘坐标系 |
| `action_server_timeout` | `5.0` | Action 服务器超时时间（秒） |

### 4.2 话题触发策略执行

策略控制器通过以下 ROS2 话题进行控制，无需启动 RViz 即可通过命令行触发。

**策略控制话题一览：**

| 话题 | 类型 | 方向 | 说明 |
|------|------|------|------|
| `route_strategy/start` | `std_msgs/Empty` | 订阅 | 启动策略执行 |
| `route_strategy/switch_branch` | `std_msgs/Int32` | 订阅 | 切换分支路线（2 或 3） |
| `route_strategy/cancel` | `std_msgs/Empty` | 订阅 | 取消策略执行 |
| `route_strategy/state` | `std_msgs/String` | 发布 | 当前策略状态（反馈） |
| `route_strategy/active_route` | `std_msgs/Int32` | 发布 | 当前活跃路线编号（反馈） |

**启动策略执行：**

```bash
ros2 topic pub /route_strategy/start std_msgs/msg/Empty "{}" --once
```

该命令触发策略从 `route_1` 开始执行，依次导航到 `route_1` 最后一个点，然后进入等待切换状态。

**切换分支路线：**

```bash
# 切换到分支路线 2
ros2 topic pub /route_strategy/switch_branch std_msgs/msg/Int32 "data: 2" --once

# 切换到分支路线 3
ros2 topic pub /route_strategy/switch_branch std_msgs/msg/Int32 "data: 3" --once
```

在等待切换状态（`WAITING_SWITCH`）下，发送此命令可指定走哪条分支路线。如果超时未收到切换指令，将使用默认分支路线（`default_branch_route` 参数，默认值 2）。

**取消策略执行：**

```bash
ros2 topic pub /route_strategy/cancel std_msgs/msg/Empty "{}" --once
```

取消当前正在执行的策略，重置为 IDLE 状态。

**查看策略状态：**

```bash
ros2 topic echo /route_strategy/state
```

状态值：`IDLE`、`ROUTE_1`、`WAITING_SWITCH`、`TRANSITION`、`BRANCH_ROUTE`、`ROUTE_4`、`COMPLETED`、`FAILED`

**查看当前活跃路线：**

```bash
ros2 topic echo /route_strategy/active_route
```

### 4.3 话题触发导航到指定路线

不依赖策略控制器，可以直接通过话题向 Nav2 发送导航目标点，让小车导航到路线列表中指定的点位。

**通过 Nav2 Action 发送导航目标（推荐）：**

```bash
ros2 action send_goal /navigate_to_pose nav2_msgs/action/NavigateToPose \
  "{pose: {header: {frame_id: 'map'}, pose: {position: {x: 1.5, y: -2.0, z: 0.0}, orientation: {x: 0.0, y: 0.0, z: 0.0, w: 1.0}}}}"
```

将 `x`、`y`、`orientation` 替换为路线文件中对应的点位坐标即可。

**通过 `/goal_pose` 话题发送导航目标：**

```bash
ros2 topic pub /goal_pose geometry_msgs/msg/PoseStamped \
  "{header: {frame_id: 'map'}, pose: {position: {x: 1.5, y: -2.0, z: 0.0}, orientation: {x: 0.0, y: 0.0, z: 0.0, w: 1.0}}}" --once
```

**示例：从路线文件中提取点位坐标后发送**

假设 `global_routes.yaml` 中 route_1 的第一个点：

```yaml
- x: -1.696
  y: -2.271
  orientation: {x: 0.0, y: 0.0, z: 0.006442, w: 0.999979}
```

发送命令：

```bash
ros2 action send_goal /navigate_to_pose nav2_msgs/action/NavigateToPose \
  "{pose: {header: {frame_id: 'map'}, pose: {position: {x: -1.696, y: -2.271, z: 0.0}, orientation: {x: 0.0, y: 0.0, z: 0.006442, w: 0.999979}}}}"
```

```bash
  map:=maps/real/new_map.yaml \
  routes_file:=maps/real/new_routes.yaml

# ===== 策略触发 =====
# 8. 启动策略执行
ros2 topic pub /route_strategy/start std_msgs/msg/Empty "{}" --once

# 9. 切换分支路线到 route_2
```

### 5.2 TF 坐标系检查

TF 树是否连通是导航正常运行的前提：

```bash
# 查看完整 TF 树
ros2 run tf2_tools view_frames

# 检查 odom_combined → base_footprint
ros2 run tf2_ros tf2_echo odom_combined base_footprint

# 检查 base_footprint → laser_link
ros2 run tf2_ros tf2_echo base_footprint laser_link

# 建图/导航模式下，检查 map → odom_combined
ros2 run tf2_ros tf2_echo map odom_combined
```

**正常 TF 链：**

```
map → odom_combined → base_footprint → base_link → laser_link
                                         └→ gyro_link
```

**静态 TF 参数说明（在 `real_robot.launch.py` 中配置）：**

| 变换 | x | y | z | 说明 |
|------|---|---|---|------|
| base_footprint → base_link | 0.41 | 0.12 | 0 | 底盘中心到车身中心 |
| base_footprint → gyro_link | 0 | 0 | 0 | 底盘中心到 IMU |
| base_footprint → laser_link | 0 | 0 | 0.18 | 底盘中心到激光雷达 |

### 5.3 Nav2 生命周期检查

导航启动后，确认各 Nav2 节点已激活：

```bash
# 检查全局规划器
ros2 lifecycle get /planner_server

# 检查控制器
ros2 lifecycle get /controller_server

# 检查行为树导航器
ros2 lifecycle get /bt_navigator

# 检查地图服务器
ros2 lifecycle get /map_server
```

正常状态均为 `active [3]`（状态编号：1=unconfigured, 2=inactive, 3=active）。

### 5.4 地图保存


**保存成功后会生成两个文件：**
