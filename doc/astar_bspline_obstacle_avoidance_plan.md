# A* + 3 阶 B 样条局部避障预留方案

本文档保存后续避障控制逻辑的设计思路。当前代码阶段只实现纯追踪跟踪全局路径，本文方案暂不接入控制器。

## 总体思路

后续控制器将先根据全局路径选择一个局部目标点，再在 Nav2 局部代价地图中用 A* 搜索从小车当前位置到该局部目标点的可行路径。搜索成功后，用 3 阶 B 样条平滑局部路径，最后让纯追踪控制器跟踪这条新的局部路径。

控制链路：

```text
全局路径 -> 选择局部目标点 -> 局部 costmap A* -> 3 阶 B 样条平滑 -> 纯追踪跟踪
```

## 局部目标点选择

起点为小车当前位置，终点从全局路径上动态选取：

1. 查找小车当前位置在全局路径上的最近点。
2. 根据当前速度计算向前搜索距离：

   ```text
   target_distance = clamp(v * target_time, min_target_distance, max_target_distance)
   ```

3. 从最近路径点开始沿全局路径累计距离。
4. 第一个累计距离大于 `target_distance` 的点作为局部目标点。
5. 如果全局路径剩余长度不足，则使用全局路径终点。

建议默认值：

| 参数 | 建议值 | 说明 |
| --- | --- | --- |
| `local_replan_frequency` | `5.0 - 10.0Hz` | 固定低频局部重规划 |
| `target_time` | `1.0s` | 根据速度向前看的时间 |
| `min_target_distance` | `0.40m` | 最小局部目标距离 |
| `max_target_distance` | `1.20m` | 最大局部目标距离，需小于局部 costmap 有效范围 |

## A* 搜索

A* 在 `local_costmap` 对应的 `Costmap2D` 中搜索：

- 起点：当前机器人位置转换到 costmap 栅格坐标。
- 终点：局部目标点转换到 costmap 栅格坐标。
- 邻域：优先使用 8 邻域，允许斜向运动。
- 启发函数：欧氏距离。
- 栅格通过性：代价值大于等于 `INSCRIBED_INFLATED_OBSTACLE` 或 `LETHAL_OBSTACLE` 时不可通行。
- 未知区域：默认不可通行，后续可通过参数切换。
- 代价函数：

  ```text
  f(n) = g(n) + h(n)
  g(n) = 距离累计代价 + costmap_cost_weight * normalized_cost
  h(n) = 到目标点的欧氏距离
  ```

为了避免搜索结果过度偏离全局路径，可增加一个较小的全局路径偏好项：

```text
global_path_bias * distance_to_global_path
```

该项权重不能过大，否则会削弱避障能力。

## 3 阶 B 样条平滑

A* 输出的是栅格折线路径，直接跟踪容易产生转向抖动，因此需要平滑：

1. 将 A* 栅格路径转换为世界坐标路径。
2. 对路径点做抽稀，避免控制点过密。
3. 使用 3 阶均匀 B 样条生成平滑曲线。
4. 按固定分辨率重新采样，例如 `0.03 - 0.05m`。
5. 根据相邻采样点方向生成路径点 yaw。

3 阶 B 样条平滑后不一定仍然完全避障，因此必须做碰撞复查。

## 碰撞检查

平滑路径生成后，需要逐点检查：

- 每个采样点是否仍在 costmap 内。
- 采样点代价值是否低于障碍阈值。
- 可选：按机器人半径或 footprint 检查邻域栅格。

处理策略：

- 如果平滑路径安全，使用 B 样条路径。
- 如果平滑路径不安全，但原始 A* 路径安全，回退使用原始 A* 路径。
- 如果 A* 路径也不可用，按失败策略回退跟踪全局路径。

## 失败回退策略

你已确认的策略是：A* 搜索失败时回退跟踪全局路径。

建议后续实现时增加保护：

- 回退时不破坏上一条可用路径，必要时由上层恢复行为或停车逻辑接管。
- 打印节流告警，提示局部避障失败。
- 连续失败次数超过阈值后，可选择返回控制失败，让 Nav2 触发恢复行为。

## 与纯追踪控制器融合方式

后续融合时，纯追踪控制器不需要重写，只需要把“被跟踪路径”的来源改成：

```text
如果局部避障路径有效 -> 跟踪局部避障路径
否则 -> 跟踪全局路径
```

也就是说，当前阶段实现的前视点搜索、倒车判断、转向角计算、速度控制都可以继续复用。
